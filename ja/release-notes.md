* [Console] 送信リクエスト失敗件について照会できる機能が追加されました。
    * AS-IS : SMS、MMS送信時、失敗した場合失敗レスポンスは受信するがConsoleで照会できない
    * TO-BE : SMS、MMS送信時、失敗した場合リクエスト状態を失敗に変更して照会できるように変更
* [API] 複数の受信者リストでの送信時、有効性検査正常件のみ送信されるようにロジックが変更されました。
    * AS-IS : 複数の受信者リストの送信リクエストが失敗した場合、失敗した受信者以降の受信者は未送信
    * TO-BE : すべての受信者に送信し、responseで受信者別送信結果を提供。送信失敗があってもリクエスト結果は成功。<br/>
      詳細は[[APIガイド](./api-guide/)]をご参照ください。
* 料金精算方式が変更されました。
    * AS-IS : メッセージ送信リクエスト時間を基準に課金
    * TO-BE : メッセージ送信結果をレスポンス受信する時間で課金

#### バグ修正
* [API] MMS添付ファイル送信時、存在しない添付ファイルの場合サーバーエラーが発生するバグが修正されました。
    * AS-IS : 存在しない添付ファイルIDで送信する場合サーバーエラー失敗レスポンス
    * TO-BE : MMS添付ファイル送信で存在しない添付ファイルがある場合、失敗原因をエラーメッセージで提供
* [Console] 大量送信でCSVテンプレートファイル空白バグが修正されました。
    * AS-IS : CSVテンプレートファイルでフィールド間に,基準で空白がある場合パースエラー
    * TO-BE : CSVテンプレートファイルでフィールド間に,基準で空白がある場合正常動作

### 2016.08.04
#### バグ修正
* [API] SMS発信番号が携帯電話認証で登録された場合、MMS送信ができないバグが修正されました。
    * 現象 : 携帯電話認証で保存される発信番号が国際番号形式（82-1x-xxxx-xxxx）で保存され、これによりMMS送信ができない問題があり
    * 解決 : 国際番号で保存されていた方式を韓国携帯電話番号形式（01x-xxxx-xxxx）に変更して保存されるよう修正
* [API] countryCodeを空白で入力時、メッセージ送信ができないバグが修正されました。
    * 現象 : countryCodeに空の値（null、""）送信時、レスポンスは成功として表示されるが正常に送信されない現象
    * 解決 : countryCodeに空の値が渡された場合、国番号82がデフォルトで設定されるよう修正
